skgstat_uncertainty.chapters.learn_geostatistics.__bin_intro = Multiline-String#
Show Value
 1To learn about spatial correlation in the dataset, that is not dependent on the location, we
 2need to calculate separating distances between observations. This way, we can correlate observation similarity
 3to the distance between them.
 4
 5A variogram being a statistical property of the sample (2nd moment), we need to aggregate the similarity
 6in observations at different separating distance lag classes. Therefore, for all combinations of point pairs,
 7their distance is calculated and saved as a distance matrix.
 8This enables us to inspect increasing variablity with increasing distance as a general pattern.
 9
10Using Python's `skgstat` library, we can choose between many different methods that will do the binning for us.
11Some of them need the number of lag classes as a parameter, others will estimate this automatically.
skgstat_uncertainty.chapters.learn_geostatistics.__est_intro = Multiline-String#
Show Value
1After aggreagting all pairs of observations into distinct lag classes, we need an estimator to express (dis-)similarity
2between the members in each group. We use so called semi-variance estimators, which are all largely built on the
3univariante variance.
4Thus, we are comparing how the varaince in observation residuals changes with increasing distance.
5
6Choose one of the pre-defined estimators and inspect the changes to the overall structure of the empirical variogram.
skgstat_uncertainty.chapters.learn_geostatistics.__fit_method_intro = Multiline-String#
Show Value
 1As of now, the empirical variogram is finished. You can already learn a lot from this figure.
 2The most important question is if the data actually bears a spatial auto-correlation. Does the semi-variance
 3increase with distance?
 4
 5If not, it's not really worth to continue using geostatistics, as anything from here on, will rely on the empirical
 6variogram and how well it captures the spatial structure of the data.
 7
 8The empirical variogram is characterized by three different parameters:
 9
10  * **sill** is the maximium semi-variance, which is approached asymtotically by the variogram
11  * **range** is the distance at which the sill is reached. Beyond this distance, the sample is spatially uncorrelated
12  * **nugget** is the y-axis intercept. It's ratio to sill describes the un-explainable share of the sample variance which is nugget + sill
13
14It is generally possible in `skgstat` to turn the use of a nugget effect completely off, by setting the `use_nugget` parameter.
15You can estimate the parameters by eye, or capture them more systematically by fitting a theoretical variogram function
16to the empirical variogram. This will also let you use the model parameterization in more sophisticated geostatistical applications.
17Using `skgstat` you can choose between different methods for fitting.

skgstat_uncertainty.chapters.learn_geostatistics.main_app(api: skgstat_uncertainty.api.API, **kwargs)#

Tutorial chapter about geostatistics. This streamlit application can be run on its own or embedded into another application. This application is the main entrypoint into the SciKit-GStat Uncertainty geostatistical applications and a generic introduction and tutorial about geostatistics. There are no uncertainty considerations embedded into this application. The user is guided from the selection of a pre-defined spatial dataset through all steps necessary to estimate an empirical variogram, then select and parameterize a theoretical variogram model and finally apply this model using one of the implemented kriging algorithms.

Parameters:

api (skgstat_uncertainty.api.API) – Connected instance of the SciKit-GStat Python API to interact with the backend.
